Second deep-purge findings
--------------------------
The first rename removed the visible full product name, but a broader source audit
found remaining retired media-mode aliases, retired webcam/media configuration
keys, and retired generated-sound identifiers. Those compatibility aliases were
removed because this build intentionally targets a disposable test database and
does not need migration support from the former brand.

A permanent validator was added:
- tools/hui_brand_purge_doctor.py

It checks:
- file and directory names
- UTF-8, UTF-16 little-endian, UTF-16 big-endian, and Latin-1 views
- Unicode normalization
- URL-, HTML-, and backslash-encoded text
- binary metadata visible through decoded byte views
- nested ZIP member names and contents

The remaining lowercase word "echo" appears only where it has a technical
meaning, including acoustic echo cancellation, shell output commands, terminal
input mode, WebRTC loopback diagnostics, and message-delivery acknowledgments.
No remaining occurrence is a product or subsystem brand.
